# Xournal++ Visual Notes

This directory contains the visual documentation for NucleOS.

Xournal++ is used as a visual engineering notebook.

## Purpose

The goal is to document ideas visually through:

- diagrams
- examples
- handwritten notes
- sketches

## Directory Structure

diagrams/

Main technical diagrams.

examples/

Visual examples that explain concepts.

notes/

Handwritten technical notes.

sketches/

Rough ideas and early visual drafts.

exported-pdfs/

PDF exports of the Xournal++ files.

## File Rule

Every important `.xopp` file should have a readable `.pdf` export.

The `.xopp` file is the editable source.

The `.pdf` file is the readable version for GitHub visitors.

## Example

Editable file:

diagrams/day-03-minimal-kernel-structure.xopp

Readable export:

exported-pdfs/diagrams/day-03-minimal-kernel-structure.pdf

## Why Export to PDF?

GitHub does not preview Xournal++ `.xopp` files directly.

PDF exports make the visual notes easier to open, read, and review.

## Current Visual Workflow

Obsidian = textual documentation

Xournal++ = visual documentation

VS Code = source code

GitHub = public evidence
